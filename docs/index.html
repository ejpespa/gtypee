npm install

3 Authentication

gtypee supports two authentication modes. Choose the one that fits your use case.

OAuth 2.0

For personal accounts & accessing your own data

Step 1: Create a Google Cloud Project
  1. Go to Google Cloud Console
  2. Create a new project (e.g., gtypee)
  3. Go to APIs & Services → OAuth consent screen
  4. Select External user type
  5. Fill in required fields, add your email as a Test user
Step 2: Enable APIs
API Services
Gmail API gmail
Google Calendar API calendar
Google Drive API drive, docs, sheets, slides
Google Tasks API tasks
Google Forms API forms
People API contacts, people
Google Chat API chat
Google Classroom API classroom
Apps Script API appscript
Step 3: Create OAuth Credentials & Authenticate

Create credentials.json in your config directory:

{
  "clientId": "YOUR_CLIENT_ID.apps.googleusercontent.com",
  "clientSecret": "YOUR_CLIENT_SECRET"
}
# Browser flow
gtypee auth add --email you@gmail.com

# Manual flow (no browser auto-open)
gtypee auth add --email you@gmail.com --manual

# Remote/headless (SSH, server)
gtypee auth add --email you@gmail.com --remote --step 1

Service Account

For Workspace admin, automated scripts, headless servers

# Import the service account key
gtypee auth add-sa --key-file /path/to/sa-key.json
gtypee auth set-default-sa --email sa@project.iam.gserviceaccount.com

# Use with impersonation
gtypee --sa sa@project.iam.gserviceaccount.com --impersonate user@domain.com gmail labels
gtypee --impersonate user@domain.com drive ls
Requires domain-wide delegation enabled and scopes authorized in the Google Admin Console under Security → API Controls.

Flag Description

Usage Scenarios

Real-world automation examples you can use today

Onboarding a New Employee 
# 1. Create the user
gtypee workspace user create \
  --email john.doe@company.com \
  --first-name John --last-name Doe \
  --password "Temp@12345" --org-unit "/Engineering"

# 2. Add to groups
gtypee workspace group add-member \
  --group engineering@company.com \
  --email john.doe@company.com --role MEMBER

# 3. Generate backup codes for 2FA
gtypee workspace user generate-backup-codes \
  --email john.doe@company.com

# 4. Verify
gtypee workspace user list --json | \
  jq '.[] | select(.primaryEmail == "john.doe@company.com")'
Offboarding an Employee 
# 1. Suspend the account immediately
gtypee workspace user suspend --email former@company.com

# 2. Remove from all groups
gtypee workspace group list --json | jq '.[].email' | while read group; do
  gtypee workspace group remove-member --group "$group" --email former@company.com
done

# 3. Delete after hold period (30+ days)
gtypee workspace user delete --email former@company.com --force
Security Audit & Login Review 
# Failed login attempts
gtypee workspace report logins --days 30 --json | \
  jq '.[] | select(.success == false)'

# Logins from unusual locations
gtypee workspace report logins --days 7 --json | \
  jq '.[] | select(.country != "US") | {userEmail, ipAddress, country}'

# Admin activity audit
gtypee workspace report admin --days 7 --json > admin-audit.json

# Export all users to CSV
gtypee workspace user list --json | \
  jq -r '.[] | [.primaryEmail, .name.givenName, .orgUnitPath] | @csv' > users.csv
